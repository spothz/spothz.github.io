Блог на GitHub через Hugo.

Если вы как и я только установили git обязательной процедурой будет зарегистрировать пользователя это делается один раз.

git config --global user.email "spot7106@gmail.com"
git config --global user.name " spothz"

список основных команд git

Создаём два репозитория один простой для разработки с любым названием. второй GitHub pages при создании указать и добавить readme.md называем ==username.gitgub.io== клонирует ранее созданный репозиторий разработки:

git clone git@github.com:spothz/spot.blog.git
cd spot.blog

Установка hugo:

переходим в репозиторий разработки и устанавливаем командой.

cd spot.blog

Debian

sudo apt install hugo

Arch

sudo pacman -Sy hugo

Termux

sudo pkg install hugo

Создаём новый сайт: 

hugo new site . --force

У Hugo есть темы. Я буду использовать тему Monochrome в этой статье. Для того, чтобы применить тему, добавляем submodule:

git submodule add https://github.com/kaiiiz/hugo-theme-monochrome.git themes/monochrome

И добавляем ее в конфиг Hugo:

echo 'theme = "monochrome"' >> hugo.toml

Создаем первый пост:

my.dev.spot.blog.md

hugo new post/greeting.mdmy.dev.spot.blog.md

После открываем файл

nano content/post/greeting.md

и видим следующее:

+++ title = 'my.dev.spot.blog.md' date = 2023-12-06T22:43:16+07:00 draft = true +++ Изменим под наши нужды:

+++ title = 'название поста,заголовок' date = 2025-01-06T20:13:16+10:00 draft = true tags = ['openwrt', 'блог'] categories = ['приветствие', 'блог'] +++

Тут любой тест идущий в пост печатаем в формате markdown. Разделителями +++ мы добавляем секцию информации о посте. title — название, заголовок date — дата draft — черновик tags — теги categories — категории

Создаём скрипт для деплоя: 

nano deploy.sh

и приводим к виду:

#!/bin/sh

set -e

printf "\033[0;32mДеплой на Github!..\033[0m\n"

msg="rebuilding site $(date)"
if [ -n "$*" ]; then
	msg="$*"
fi

echo ""
echo ""
echo "Коммитим изменения в $(pwd)"
hugo -D

cd public

echo ""
echo ""
echo "Коммитим изменения в $(pwd)"
git add .
git commit -m "$msg"
git push origin

cd ..
echo ""
echo ""
echo "Коммитим изменения в pwd $(pwd)"
git add .
git commit -m "$msg"
git push origin

Выдаём права:

chmod +x deploy.sh

Проверяем работу сайта запустив локаторный сервер: 

hugo server -D

переходим по ссылке

http://localhost:1313/

Выполняем команды:

git add .
git commit -m "Deploy site"
git push

Приступаем к публикации и деплой:

Клонируем spothz.github.io и переходим в него:

git clone git@github.com:spothz/spothz.github.io.git
cd spothz.github.io.git

Добавляем submodule:

 git submodule add git@spothz.github.io.git public

Он добавляет наш сайт в директорию public репозитория pages. Обновляем модули:

git submodule update

Делаем публикацию:

git add .
git commit -m "Deploy site"
git push

Переходим в репозиторий разработки:

cd spot.blog

выполняем:

./deploy.sh

После ожидаем 10 мин и заходим по адресу https://spothz.github.io сайт должен быть опубликован.
